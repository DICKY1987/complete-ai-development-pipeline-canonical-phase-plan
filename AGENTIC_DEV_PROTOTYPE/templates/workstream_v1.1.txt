# WORKSTREAM_V1.1 Prompt Template

This template defines the structure for AI-assisted development prompts
following the Game Board Protocol.

## Template Structure

```
================================================================================
WORKSTREAM_V1.1: AI-ASSISTED DEVELOPMENT PROTOCOL
================================================================================

PHASE_ID: {phase_id}
PHASE_NAME: {phase_name}

This is a structured development workstream following the Game Board Protocol.
You are an AI development assistant tasked with executing this phase.

OBJECTIVE:
--------------------------------------------------------------------------------
{objective}

Your task is to implement this objective while following all specifications,
passing all acceptance tests, and delivering all required artifacts.

EMBEDDED_SPECIFICATION_CONTEXT:
--------------------------------------------------------------------------------
{embedded_specs}

PRE_FLIGHT_CHECKS:
--------------------------------------------------------------------------------
Execute these checks BEFORE beginning implementation:

{pre_flight_checks}

FILE_SCOPE:
--------------------------------------------------------------------------------
You may ONLY create or modify the following files:

{file_scope}

DO NOT modify any files outside this scope.

DELIVERABLES:
--------------------------------------------------------------------------------
You must produce the following artifacts:

{deliverables}

ACCEPTANCE_TESTS (Completion Criteria):
--------------------------------------------------------------------------------
Your implementation MUST pass ALL of these tests:

{acceptance_tests}

DEPENDENCIES:
--------------------------------------------------------------------------------
{dependencies}

================================================================================
EXECUTION_METADATA:
--------------------------------------------------------------------------------
Estimated Effort: {effort_hours} hours
Risk Level: {risk_level}

NOTES:
{notes}

================================================================================
BEGIN IMPLEMENTATION
================================================================================

Follow the WORKSTREAM_V1.1 protocol:
1. Run ALL pre-flight checks
2. Create/modify ONLY files in scope
3. Deliver ALL specified artifacts
4. Pass ALL acceptance tests
5. Document your work

Good luck! Let's build something great.
```

## Usage

The prompt renderer (`src/prompt_renderer.py`) uses this template structure
to generate AI-optimized prompts from phase specifications.

## Key Sections

- **OBJECTIVE**: Clear, measurable goal for the phase
- **EMBEDDED_SPECIFICATION_CONTEXT**: Referenced spec sections (optional)
- **PRE_FLIGHT_CHECKS**: Validation before starting work
- **FILE_SCOPE**: Explicit list of modifiable files
- **DELIVERABLES**: Required artifacts
- **ACCEPTANCE_TESTS**: Completion criteria (must all pass)
- **DEPENDENCIES**: Required prerequisite phases
- **EXECUTION_METADATA**: Effort estimates and risk level

## Design Principles

1. **ASCII-only**: No unicode characters for maximum compatibility
2. **Clear structure**: Explicit sections with visual separators
3. **Action-oriented**: Commands and expectations clearly stated
4. **Testable**: Acceptance criteria are programmatic tests
5. **Scoped**: File modifications are explicitly bounded
