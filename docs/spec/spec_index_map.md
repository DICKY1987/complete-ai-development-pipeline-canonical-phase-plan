## Semantic Mapping Rules

The mapping logic uses the following semantic rules:

### Module Assignment
- `IDX-DB-*` → `src/pipeline/db.py` (database operations)
- `IDX-PROMPT-*` → `src/pipeline/prompts.py` (AI prompts)
- `IDX-TOOL-*` → `src/pipeline/tools.py` (external tool adapters)
- `IDX-WORKTREE-*` → `src/pipeline/worktree.py` (git worktree management)
- `IDX-STATE-*` → `src/pipeline/db.py` (state machine)
- `IDX-SCHEMA-*` → `schema/schema.sql` (database schema)
- `IDX-CB-*` → `src/pipeline/circuit_breakers.py` (circuit breakers)
- `IDX-RECOVERY-*` → `src/pipeline/recovery.py` (crash recovery)

### Phase Assignment
- **PH-01**: Spec mapping, index scanning, module stubs
- **PH-02**: Database, state machine, CRUD operations
- **PH-03**: Tool adapters, profiles, integration
- **PH-04**: Orchestration, scheduling, execution
- **PH-05**: Circuit breakers, recovery, observability
- **PH-06**: Bundles, worktrees, full pipeline

### Version Assignment
- **v1.0**: Core functionality (IDX numbers 01-50)
- **v2.0**: Enhanced features (IDX numbers 51-99)
- **v3.0**: Advanced features (IDX numbers 100+)
