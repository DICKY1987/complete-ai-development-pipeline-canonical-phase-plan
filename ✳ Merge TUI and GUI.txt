  Root Cause

  The Textual framework's CSS parser does not support standard CSS features like:
  - :root pseudo-class selector
  - CSS custom properties (variables declared with --variable-name)
  - var() function for using CSS variables

  The original code attempted to use these standard CSS features, which caused the parser to fail
  with: "Expected selector or end of file (found ':root {\n')".

  Fixes Applied

  1. Main CSS Issue (tui_app/main.py:86-136)
  - Removed :root { } block and CSS variable declarations
  - Changed from var(--surface) syntax to direct palette value substitution using f-strings
  - All color values are now injected directly from palette.surface, palette.text, etc.
